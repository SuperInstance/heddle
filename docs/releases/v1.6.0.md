# Heddle v1.6.0

Release date: 2026-05-18

Heddle v1.6.0 continues the core boundary cleanup from v1.5.0. This release
focuses on three shared foundations: observability, LLM provider selection, and
authentication.

The main theme is making the lower-level surfaces easier to extend without
copying one-off helpers or leaking provider-specific behavior through the rest
of the system.

## Highlights

### Observability now has clearer domain services

Conversation activity projection, tool activity summaries, trace semantics, and
trace summary behavior now live under a more explicit observability structure.

Why this matters:

- TUI and control-plane surfaces have a cleaner place to get activity views
- trace summarization is easier to test and extend
- observability behavior no longer depends on one large function-heavy module

### LLM provider handling now uses provider adapters

OpenAI and Anthropic behavior moved into provider-specific adapter folders with
model policy, provider inference, and registry ownership separated from callers.

Why this matters:

- adding future providers has a clearer path
- provider-specific request/response codecs are easier to inspect
- runtime and chat-engine callers resolve models through a shared provider
  boundary instead of importing provider helpers directly

### Auth is now a first-class service boundary

OpenAI OAuth behavior and provider credential persistence now use class-based
services, shared `types.ts` contracts, and zod-backed persisted auth schemas.

Why this matters:

- stored auth data is validated at the repository boundary
- OAuth login, refresh, credential creation, and account parsing have one owner
- CLI, runtime, and LLM callers use the same credential path
