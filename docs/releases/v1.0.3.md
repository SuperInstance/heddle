# Heddle v1.0.3

Release date: 2026-05-09

Heddle v1.0.3 is a patch release focused on making fresh sessions understand project intent faster and more predictably.

The main theme is better startup context: projects can now provide a Heddle-native instruction file, and Heddle's own public docs now give coding agents a shorter, clearer discovery path before they make implementation choices.

## Highlights

### HEDDLE.md project instructions

Heddle now looks for one project instruction file at startup. The default priority is:

1. `HEDDLE.md`
2. `AGENTS.md`
3. `CLAUDE.md`

Only the first non-empty default file is loaded, so projects can provide Heddle-specific instructions without spending context on multiple overlapping agent files. Explicit `agentContextPaths` in `heddle.config.json` still wins for projects that need custom paths or intentionally want multiple instruction files.

Why this matters:

- fresh Heddle sessions can start from the project's own operating guidance
- Heddle-specific instructions no longer have to be hidden inside another agent's file
- context usage stays bounded by default

### Project guidance is easier for agents to follow

The public docs now include a concise project posture guide for coding agents, plus a clearer top-level docs map.

Why this matters:

- agents have a shorter path to Heddle's product posture, architecture map, and invariants
- contributors can tell which docs are always useful and which docs are task-specific
- public repo guidance stays separate from optional private workspace notes

### Project context is appended last

Startup project instructions are appended after the memory context block. This keeps the stable system prompt and memory framing earlier, while placing project-specific dynamic instructions at the end where they are easiest to inspect and evolve.

## Other Changes

- Added Heddle's own root `HEDDLE.md` with a pointer to `docs/agent-context.md`.
- Documented project instruction discovery in the README, chat guide, and config reference.
- Added regression coverage for project instruction selection, config override behavior, and prompt-context ordering.
