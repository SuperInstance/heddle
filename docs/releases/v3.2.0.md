# Heddle v3.2.0

Release date: 2026-06-07

Heddle v3.2.0 adds the first opt-in browser automation foundation for
programmatic agents. The release gives browser work a dedicated core domain,
toolkit surface, safety policy, and examples without making browser control part
of the default CLI agent path.

## Highlights

### Browser automation foundation

Heddle now includes a dedicated browser domain under `src/core/browser`.

Why this matters:

- browser profiles, sessions, snapshots, evidence, and policy checks now have a
  clear owner
- Playwright-backed execution can be used by opt-in callers through the browser
  driver boundary
- the browser implementation is isolated enough to evolve or be removed without
  entangling the default coding-agent runtime

### Browser research toolkit

Programmatic callers can now create a browser research toolkit with browser
tools for opening pages, taking snapshots, clicking snapshot refs, capturing
screenshots, and closing sessions.

Why this matters:

- agents can work against a traceable browser surface instead of ad hoc scripts
- headed and headless examples are available for local validation
- browser artifacts are captured as reviewable evidence for debugging and
  product evaluation

### Safety hardening

This release keeps browser automation behind explicit caller policy.

Why this matters:

- domain allowlists and forbidden-action text gate browser navigation and clicks
- snapshot refs are revoked after navigation so stale refs cannot be reused on a
  different page state
- click redirects are checked before loading disallowed targets
- browser profile locks are released on failure paths, including failed open and
  close operations

### Optional Playwright dependency

Playwright remains an opt-in runtime dependency for browser users.

Why this matters:

- importing the package root does not require Playwright to be installed
- callers who do not use browser tools are not affected by browser automation
  dependencies
- browser users can install Playwright or provide a custom driver factory

## Upgrade Notes

- No migration is required for existing CLI or programmatic non-browser users.
- Browser automation is experimental and opt-in.
- Callers using Heddle's default Playwright browser driver should install
  `playwright` in their runtime environment.
- The browser research toolkit is intended for controlled research flows with
  explicit allowlists and operator-reviewed evidence.
