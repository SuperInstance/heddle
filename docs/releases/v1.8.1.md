# Heddle v1.8.1

Release date: 2026-05-20

Heddle v1.8.1 continues the `1.8.x` web-v2 refactoring line with a focused
control-plane UI polish pass. The release keeps the public product surface small
while moving the new browser shell toward standard shadcn primitives and a more
consistent design language.

## Highlights

### Web-v2 sidebar now uses shadcn-style primitives

The v2 shell now uses a dedicated sidebar primitive with standard collapse,
tooltip, sheet, and mobile-aware behavior instead of one-off sidebar wiring.

Why this matters:

- sidebar behavior follows a more familiar frontend pattern
- future navigation changes can build on a reusable primitive
- the web-v2 shell keeps its dependency boundary inside `src/web-v2`

### Visual tokens are more consistent

Panel borders, popovers, navigation rows, sidebar colors, and control sizing now
share the same web-v2 token language.

Why this matters:

- new UI slices have a clearer visual baseline to copy
- navigation and settings surfaces feel less ad hoc
- design decisions are captured in the v2 frontend instead of scattered styles

### Browser checks cover the sidebar interaction

The web-v2 browser integration test now checks sidebar collapse and expansion in
addition to the existing route and shell coverage.

Why this matters:

- the sidebar interaction is guarded in CI
- accessibility landmarks stay visible to Playwright and assistive tooling
- route-backed v2 navigation remains covered while the UI is still evolving

### shadcn installs now target web-v2

The shadcn component config now points new components at `src/web-v2` so future
component additions do not accidentally land in the legacy web tree.
