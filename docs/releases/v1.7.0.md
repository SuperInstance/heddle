# Heddle v1.7.0

Release date: 2026-05-18

Heddle v1.7.0 continues the core boundary cleanup with a focus on memory,
review, trace, command, and tool infrastructure.

The main theme is making core modules easier to inspect and extend by replacing
loose helper-function seams with clearer class-backed boundaries, local
`types.ts` contracts, repository ownership for file I/O, and small README
placement rules where they help future contributors.

## Highlights

### Memory now has a clearer service boundary

Workspace memory now has typed service and repository ownership for catalog,
maintenance, notes, visibility, validation, and persisted records.

Why this matters:

- memory behavior is easier to test without reaching through unrelated tools
- persisted memory files are validated at the boundary
- the maintainer prompt is easier to review and keeps stable markdown
  instructions before dynamic context for better provider caching

### Review and trace have explicit class APIs

Review diff parsing and trace recording/formatting now use named class APIs such
as `ReviewDiffParser`, `TraceRecorder`, and `TraceConsoleFormatter`.

Why this matters:

- control-plane and CLI callers can depend on one clear boundary
- trace presentation and trace persistence are easier to evolve separately
- old one-off parser and formatter imports are removed from the public shape

### Commands and tools are grouped behind service classes

Slash commands now use parser, autocomplete, and registry classes. Tool
registration, execution, and bundle composition now have dedicated class
boundaries.

Why this matters:

- command behavior has a clearer place to live
- tool execution policy is less scattered across callers
- future toolkits can be added without copying setup logic between hosts

### Agent budget logic moved into the agent domain

Step-budget behavior now lives under `src/core/agent` instead of the generic
utility layer.

Why this matters:

- agent-loop policy is closer to the loop that owns it
- shared utilities stay smaller and more domain-neutral

## Operator Notes

- Package version is now `1.7.0`.
- Latest already-published npm version before this release was `1.6.0`.
- This release uses the real git range `v1.6.0..HEAD`.
- The normal release baseline remains `yarn build`, `yarn test`, and
  `npm pack --dry-run --cache /tmp/heddle-npm-cache`.
- Final npm publication remains a manual operator step after the tagged GitHub
  release is prepared.

## Upgrade Notes

This release intentionally continues the alpha API cleanup for lower-level
programmatic users. Prefer the class APIs documented in the programmatic use
guide when assembling custom hosts, tool surfaces, review flows, or trace
outputs.

The main CLI, terminal chat, memory commands, and browser control-plane
workflows are expected to behave the same.
