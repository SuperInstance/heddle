# Heddle v1.11.0

Release date: 2026-05-23

Heddle v1.11.0 continues the web-v2 control-plane work with a more complete
session workspace. The release wires model and reasoning controls into the v2
composer, makes selected sessions addressable by URL, tightens the approval
panel, and improves the mobile web-v2 frame for real phone testing.

## Highlights

### Web-v2 model settings are wired to the session API

The v2 composer now exposes model and reasoning effort controls backed by the
control-plane session API. Changing either setting persists through the session
controller and is covered by browser integration tests.

Why this matters:

- web-v2 can exercise the same model/reasoning choices as the rest of Heddle
- settings are no longer only local component state
- browser tests cover both UI selection and API persistence

### Session selection is reflected in the URL

The web-v2 workbench now routes selected sessions through the browser path
instead of keeping selection only in React state.

Why this matters:

- a selected session can be refreshed or shared without losing context
- session switching is easier to reason about from the route layer
- the route inventory is explicit and centralized for future v2 surfaces

### Tool approvals are more compact

The approval panel now removes duplicate metadata, places actions below the
request detail, keeps denial as the final action, and uses smaller controls for
the common approval path.

Why this matters:

- pending approvals take less vertical space in the conversation
- long shell commands stay readable instead of squeezing the action row
- the mobile approval layout remains usable on narrow screens

### Mobile web-v2 is a dedicated frame

The v2 shell now separates desktop and mobile app frames while sharing the
common navigation and workbench pieces. Mobile panels open as overlays instead
of squeezing the conversation, the composer starts at one line and grows as
needed, and the viewport is constrained to avoid horizontal drift and accidental
pinch zoom during phone testing.

Why this matters:

- mobile layout decisions live in the mobile frame instead of scattered
  conditionals
- the workbench remains stable when side panels open
- the browser status-bar color, safe-area background, and scroll containment now
  match the v2 header treatment more closely

### README includes web-v2 mobile screenshots

The root README now shows the web-v2 mobile session list, workbench, and diff
preview screenshots alongside the existing web-v2 work-in-progress control-plane
image.
